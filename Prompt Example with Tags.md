# Primary Tags
[PLAN]       – High-level planning, roadmap, architecture.
[PROGRESS]   – Update on completed tasks (roadmap/devlog).
[CODE]       – Request for new code to be written.
[DEBUG]      – Errors, crashes, unexpected behaviour.
[REFACTOR]   – Improve clarity / structure of working code.
[REVIEW]     – Ask ChatGPT to review code without rewriting.
[DOC]        – Documentation updates (README, ROADMAP, DEVLOG).
[ARCH]       – Architecture design (modules, interfaces, flow).
[TEST]       – Test plans, test cases, behavioural checks.

# Secondary Tags
[UI]         – Layouts, views, screens, navigation.
[UX]         – Behaviour, flows, user-facing interactions.
[DATA]       – Datasets, GeoJSON, storage, schemas.
[API]        – Endpoints, network syncs, API design.
[CONFIG]     – Constants, thresholds, settings, flags.
[BUILD]      – Gradle, manifests, dependency issues.
[RELEASE]    – Packaging, internal testing, Play Store prep.

# Advanced / Optional Tags
[PERF]       – Battery optimisation, background efficiency.
[SEC]        – Security, privacy, permission rules.
[SIM]        – Simulated events, mock data, fake location.
[DISCUSS]    – Open-ended brainstorming or comparison.


# Prompt Template Example

Use this template whenever messaging ChatGPT about development work:

[TAG]
**Context:**  
(Short description of what you’re doing. 1–3 sentences.)

**Goal:**  
(What outcome you want from ChatGPT.)

**Relevant Files / Code:**  
(Paste only the relevant code or section of ROADMAP.md/DEVLOG.md.)  
(If long, summarise.)

**Specific Questions or Actions:**  
- Question or task 1  
- Question or task 2  
- Question or task 3  

**Notes / Constraints:**  
(Anything ChatGPT should keep in mind: clarity, architecture rules, inversion, extraction, battery limits, etc.)
